## 개요
사용자가 코스피200 종목을 검색해 워치리스트에 추가하면, 그레이엄 방어적 투자자 기준을 단순화한 4개 지표(PER·PBR·부채비율·최근 순이익 흑자)로 판정한 결과와 최신 시세를 보여주는 개인용 도구.

## 범위

### 포함
- 종목명 한 글자 이상 입력 시 코스피200 마스터 리스트 기준 자동완성 (최대 5개)
- 자동완성에서 종목 선택 시 워치리스트에 추가
- 이미 워치리스트에 있는 종목을 다시 추가하려 하면 새로고침 여부를 묻는 확인 팝업
- 워치리스트 목록: 종목명, 현재가, 등락률, 총점(4개 중 만족 개수)
- 종목 상세: 4개 기준 각각의 통과/실패 여부 + 총점
- 재무데이터가 일부만 없는 경우 계산 가능한 기준만 판정해 "n개 중 m개 만족"으로 표시
- 워치리스트 항목 삭제
- 워치리스트를 다시 열 때마다 최신 시세·재무데이터로 갱신
- 로컬 저장 (새로고침해도 유지, 계정 없음)

### 제외
- 코스피200 밖 종목 지원 — 자동완성 마스터 리스트 크기·검색 속도를 다루기 쉬운 범위로 제한하기 위함 (3시간 제약)
- 전체 시장 자동 스캔·추천 — 배치 처리와 API 레이트리밋 리스크가 큼. 이번 범위는 "검색해서 판정받기"로 한정
- 그레이엄 원전의 장기(10~20년) 기준(이익 성장률, 배당 기록, 유동비율 등) — 장기 시계열 데이터를 3시간 안에 확보하기 어려움. 최근 1년 데이터로 계산 가능한 4개 지표로 단순화
- 계정/로그인, 다중 기기 동기화 — 본인 전용 로컬 도구로 범위 한정
- PER 계산 불가 시 PSR 등 대체 지표 표시 — 순이익 흑자 기준이 이미 적자 상황을 별도로 판정하므로 불필요
- 실제 매수·증권사 연동 — 리서치 도구로 범위 한정

## 시나리오

### 1. 종목 검색 자동완성
- **Given** — 워치리스트 화면, 검색창이 비어 있음
- **When** — 코스피200에 속한 종목명의 일부(예: "삼성")를 입력
- **Then** — 입력한 문자열을 포함하는 코스피200 종목명이 최대 5개까지 자동완성 목록에 나타난다

성공 기준:
- [ ] "삼성" 입력 → 자동완성 목록에 "삼성"이 포함된 코스피200 종목명이 최소 1개, 최대 5개까지 나타난다
- [ ] 코스피200에 없는 이름을 입력 → "코스피200 내에서 일치하는 종목이 없습니다" 문구가 나타난다

### 2. 종목을 워치리스트에 추가
- **Given** — 자동완성 목록에 종목이 나타난 상태
- **When** — 목록에서 종목 하나를 선택
- **Then** — 해당 종목이 워치리스트에 추가되고, 종목명·현재가·등락률·총점(4개 기준 중 만족 개수)이 표시된다

성공 기준:
- [ ] 자동완성에서 종목 선택 → 워치리스트 목록에 해당 종목명이 나타난다
- [ ] 추가된 종목 행에 현재가, 등락률, "n/4 만족" 형식의 총점이 표시된다

### 3. 이미 추가된 종목을 다시 선택
- **Given** — 워치리스트에 이미 있는 종목
- **When** — 같은 종목을 다시 검색해서 자동완성에서 선택
- **Then** — "이미 워치리스트에 있습니다. 최신 데이터로 새로고침할까요?" 확인 팝업이 나타난다
- **When** — 팝업에서 확인
- **Then** — 해당 종목의 시세·재무데이터가 다시 조회되어 갱신된다
- **When** — 팝업에서 취소
- **Then** — 아무 변화 없이 팝업이 닫히고 워치리스트는 그대로 유지된다

성공 기준:
- [ ] 중복 종목 선택 → 확인 팝업이 나타나고 워치리스트 항목 수는 늘지 않는다
- [ ] 팝업에서 확인 클릭 → 해당 종목의 표시된 수치가 최신 값으로 갱신된다
- [ ] 팝업에서 취소 클릭 → 워치리스트 내용이 이전과 동일하게 유지된다

### 4. 종목 상세 — 그레이엄 체크리스트 판정
- **Given** — 재무·시세 데이터를 모두 가져올 수 있는 종목이 워치리스트에 있음
- **When** — 해당 종목 상세를 확인
- **Then** — PER<15, PBR<1.5, 부채비율<100%, 최근 결산 순이익 흑자 4개 기준 각각의 통과·실패 여부와 "n/4 만족" 총점이 표시된다

성공 기준:
- [ ] 4개 기준을 모두 만족하는 종목 → "4/4 만족"과 4개 항목 모두 통과 표시가 나타난다
- [ ] 일부만 만족하는 종목(예: PER만 기준 초과) → 해당 기준만 실패 표시, 나머지는 통과 표시, 총점은 실제 만족 개수와 일치한다

### 5. 최근 결산 적자 종목의 PER 표시
- **Given** — 최근 결산 순이익이 적자인 종목이 워치리스트에 있음
- **When** — 해당 종목 상세를 확인
- **Then** — PER 항목에 "N/A"가 표시되고 PER 기준은 실패로 처리되며, 순이익 흑자 기준도 함께 실패로 표시된다

성공 기준:
- [ ] 최근 결산 적자 종목 → PER 항목이 "N/A"로 표시되고 PER 기준과 순이익 흑자 기준 모두 실패로 표시된다

### 6. 재무데이터 일부 누락
- **Given** — 상장 직후 등의 이유로 일부 재무 항목을 가져올 수 없는 종목이 워치리스트에 있음
- **When** — 해당 종목 상세를 확인
- **Then** — 계산 가능한 기준만 판정되어 "계산된 기준 개수 중 만족 개수"로 표시되고, 계산할 수 없는 기준에는 "데이터 없음"이 표시된다

성공 기준:
- [ ] 4개 중 1개 지표를 계산할 데이터가 없는 종목 → 나머지 3개 기준으로 판정되어 "3개 기준 중 n/3 만족"이 표시되고, 데이터 없는 1개 기준에는 "데이터 없음"이 표시된다

### 7. 워치리스트 항목 삭제
- **Given** — 워치리스트에 종목 1개 이상 있음
- **When** — 특정 종목의 삭제 버튼 클릭
- **Then** — 해당 종목이 즉시 워치리스트에서 사라진다

성공 기준:
- [ ] 종목 삭제 클릭 → 워치리스트 목록에서 해당 종목명이 사라지고 나머지 종목은 그대로 남는다

### 8. 빈 워치리스트
- **Given** — 워치리스트에 종목이 하나도 없는 상태(최초 방문 또는 모두 삭제한 뒤)
- **When** — 화면을 확인
- **Then** — "워치리스트가 비어 있습니다. 종목을 검색해 추가하세요" 안내 문구가 표시된다

성공 기준:
- [ ] 종목이 없는 상태 → 안내 문구가 표시되고 목록 영역에는 종목 행이 없다

### 9. 워치리스트 데이터 유지 및 갱신
- **Given** — 종목이 담긴 워치리스트로 페이지를 새로고침하거나 다시 방문
- **When** — 화면 로드
- **Then** — 이전에 추가했던 종목들이 그대로 남아 있고, 시세·재무데이터는 최신 값으로 다시 조회되어 표시된다

성공 기준:
- [ ] 새로고침 후 → 이전에 추가된 종목명이 모두 그대로 나타난다
- [ ] 새로고침 시점의 최신 시세(예: 전일과 다른 종가)가 반영되어 표시된다

## 불변 규칙

- 워치리스트 데이터는 사용자의 브라우저에만 저장되며 서버에 전송·저장되지 않는다
- 판정에 사용하는 4개 그레이엄 기준과 임계값(PER<15, PBR<1.5, 부채비율<100%, 순이익 흑자)은 모든 종목에 동일하게 적용된다
- 검색·자동완성 대상은 코스피200 종목으로 한정된다
